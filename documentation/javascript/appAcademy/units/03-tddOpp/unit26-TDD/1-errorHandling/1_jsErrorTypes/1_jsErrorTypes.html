1_Js Error Types

 There are 7 core Error Types:

  • SyntaxError - error in code wording
  • ReferenceError - invalid reference made
  • TypeError - Variable or Parameter is not a valid type
  • RangeError - numerical value outside of a valid range
  • InternalError - problem with the internal Javascript engine
  • EvalError - error in the global eval function
  • URIError - occurs when invalid parameters are passed to encodeURI() or decodURI()

 This lesson focus on SyntaxError, ReferenceError and TypeError

SyntaxError

 This occurs with misspelling 

funtion broken () { // Uncaught SyntaxError: Unexpected identifier
    console.log("I'm broke");
}

Or misplaced symbols and/or characters 

function broken () { // Uncaught SyntaxError: Unexpected identifier
  console.log("I'm broke");
}} // Uncaught SyntaxError: Unexpected token '}'

ReferenceError

 This occurs when a non-existent varaible is referenced or not within the available scope. 

function callPuppy() {
  const puppy = "puppy";
  console.log(pupy);
}

callPuppy(); // ReferenceError: pupy is not defined
        
function callPuppy() {
  const puppy = "puppy";
}
console.log(puppy); // ReferenceError: puppy is not defined

TypeError

A TypeError is commonly thrown for a couple of reasons:

  • When an operation cannot be performed because the operand is a value of the wrong type.
  • When you are attempting to modify a value that cannot be changed.
